PDS 채팅 모델은 **사용자 메시지에는 풍선**, **어시스턴트 응답에는 풍선 없이 본문만** 노출한다. 응답이 곧 본문이라 별도 시각 컨테이너로 가두지 않는다.
이 컴포넌트는 `ChatThread` 가 내부적으로 사용하지만, 단일 응답 미리보기·문서·테스트 용도로 단독 사용도 가능하다.

## 사용

```tsx
import { ChatAssistantMessage } from "@fluxloop-ai/pds-ui/components/chat-assistant-message";
import { renderMarkdown } from "@fluxloop-ai/pds-markdown";

<ChatAssistantMessage
  content="오늘 서울은 **맑아요**."
  renderMarkdown={renderMarkdown}
/>
```

`content` 는 문자열 또는 Anthropic 콘텐츠 블록 배열. `text` / `thinking` / `tool_use` 블록을 인식하고, 그 외는 무시.
`tool_use` 블록의 결과는 `toolResults` Map (`tool_use_id` → `ToolResultBlock`) 으로 전달한다.

마크다운 렌더는 옵셔널. `renderMarkdown` 미주입 시 plain text 로 떨어진다. 권장 구현체와 커스터마이즈 가이드는 [Chat Markdown](/docs/components/chat-markdown).

## Props

| Prop | 타입 | 기본 | 설명 |
|---|---|---|---|
| `content` | `string \| ContentBlock[]` | — | 응답 본문 |
| `toolResults` | `Map<string, ToolResultBlock>` | — | `tool_use_id` 별 결과 매핑 |
| `renderMarkdown` | `(text: string) => ReactNode` | — | 텍스트 블록 렌더러. 미지정 시 plain text 폴백 |
| `renderToolCall` | `(use, result?) => ReactNode` | `<ToolCallCard />` | tool_use 블록 커스텀 렌더 |
| `renderThinking` | `(block, phase?) => ReactNode` | `<ThinkingBlock />` | thinking 블록 커스텀 렌더 |
| `actions` | `ReactNode` | — | 응답 아래 action bar (copy/regenerate 등) |
| `className` | `string` | — | 루트 클래스 |

## 렌더 규칙

- `text` → `renderMarkdown` 결과 또는 plain text (`whitespace-pre-wrap`) 폴백
- `thinking` → `renderThinking` 결과 또는 `ThinkingBlock`
- `tool_use` → `renderToolCall` 결과 또는 `ToolCallCard` (`toolResults` 에서 매칭된 결과 전달)
- 그 외 (`image`, `tool_result`, `redacted_thinking` 등) → 무시
